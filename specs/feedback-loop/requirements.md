# Feature: Feedback Loop (P3.3)

## User Stories

### US-1: On-call SRE provides feedback on alerts

WHEN an on-call SRE receives an anomaly alert in Slack
AND reacts with ✅ (true positive) or ❌ (false positive)
THEN the system SHALL capture the reaction, associate it with the alert fingerprint and rule name, and store it for analysis.

### US-2: Team lead reviews weekly noise report

WHEN a new week begins
THEN the system SHALL post a weekly report to a configured Slack channel containing:
- Top 5 noisiest rules (highest FP count)
- Precision and recall per rule (based on available feedback)
- Trend vs previous week

### US-3: System auto-tunes thresholds based on feedback

WHEN a rule accumulates ≥N feedback samples within the retention window (30d)
AND precision drops below a configured threshold (e.g., <0.7)
THEN the system SHALL raise the zscore_threshold for that metric (tighten detection)
AND SHALL NOT lower it below the configured `min_threshold` safety floor.

## Acceptance Criteria

- [ ] Slack `reaction_added` events for ✅/❌ on alert messages are captured via webhook
- [ ] Each feedback entry is stored with: alert_fingerprint, rule_name, metric, verdict (tp/fp), timestamp
- [ ] Storage uses Redis sorted sets with TTL 30d
- [ ] Precision and recall are computable per rule over configurable time window
- [ ] Auto-tune only raises thresholds (never lowers — one-directional safety)
- [ ] Auto-tune requires minimum N samples before acting (configurable, default 20)
- [ ] Auto-tune respects `min_threshold` floor per metric (configurable, default 2.0)
- [ ] Weekly report posts to Slack with top-5 noisy rules + per-rule precision/recall
- [ ] Prometheus metrics exposed: `feedback_total{rule,verdict}`, `precision_by_rule`, `recall_by_rule`

## Out of Scope

- Replacing or modifying Alertmanager routing logic
- Building a custom Slack bot (use existing Slack Events API + reaction webhooks)
- Feedback on alerts not generated by anomaly-detection-controller
- Real-time threshold changes (batch/periodic only)
- UI for feedback visualization (Slack report is sufficient for P3.3)
